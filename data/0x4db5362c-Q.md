`r` may not be zero when `valid` is false, which could cause misunderstanding or misuse. 

    if or(eq(byte1, 0xff), eq(byte2, 0xff)) {
        valid := false
        break
    }

Should set `r` to zero when `valid` is false for consistency: 

    if or(eq(byte1, 0xff), eq(byte2, 0xff)) {
        valid := false
        r := 0
        break
    }
